Roadmap

Packal is not yet fully featured. Developments to come in future versions. These are features that we'd like to implement but haven't so that we could make Packal available. Below is a general roadmap, in no specific order, of tasks to accomplish and feature to implement. This page makes Packal more transparent, but it is also a convenient todo list for the Packal authors.

For the Packal Workflow

  • Background updates

    Update all Workflows with a single Alfred command (without the UI).

  • Workflow tracking

    An option to send information back to the Packal servers about your installed Workflows (anonymously and opt-out) to help collect information about which Workflows you currently have installed and enabled. This will be useful for tracking the popularity of Workflows, both for the authors and to allow us to improve the 'popular' Workflow pages.

  • Allow the UI to be opened in a web browser instead of the built-in viewer

    Delayed for design reasons.

  • Better Error Handling

    Currently, there is minimal error handling within the Workflow. This will be improved to make Packal faster and better.

  • Better Alfred Integration

    We know. Packal is a Workflow and resource for Alfred, but right now it feels like an adjunct application. It can be better integrated into Alfred, and that should happen.

  • Implement unit testing

    Let's write some tests to make sure that Packal functions as well as it can.

  • More configuration options

    We want Packal to look and work just how you want it to. Let's create these options.

  • Better Plist Migration

    Right now Packal migrates all necessary settings that you've overridden. But, have you changed the icons? Those aren't migrated. There are a few other unnecessary but desirable things that could be migrated.

  • Show Changelog

    Create a way to show a changelog in the updates section

  • A more elegant way to detect Internet connections

    Currently, the script to detect the Internet connection (to prevent trying to download updates when your computer can't reach the server feels like a bit of a hack. Let's replace it with something better, faster, and more elegant

  • Consolidate Packal files

    There are so many moving parts in Packal, and the logic has been to keep most everything in different scripts. Let's combine in better ways for a more elegantly written application.

  • Update scripts for Workflow authors

    Define routines to update data structures when a workflow is updated.

  • More robust integration with WHW

    Let's make better help files. Dependent on the release of the next version of WHW.

  • Shared asset handling and dependencies

    Let's allow for Workflows to declare dependencies that can be installed with Packal, allowing for faster updates and an overall smaller footprint for Workflows.

  • Replacing the bundled php5.5 webserver with something smaller and faster

    For size and performance reasons. Also, it isn't necessary with Mavericks because 10.9 has php5.4 built in. This feature is dependent on shared asset handling.

  • Faster Github Integration

    Let's reduce the number of requests to Github to make retrieving package information faster.

  • Pushing updates for your published Workflows to the Packal server from Alfred

    This one is a long way off, but it's a feature that we'd like to implement.

  • Tweaking the UI for aesthetics and usability

    Self-explanatory. Will be an on-going process, never quite done.

For the Packal Website

  • Create infrastructure for background updates

    Obviously necessary for the Workflow feature above.

  • Create infrastructure for Workflow tracking

    Necessary for feature outlined above.

  • Create an infrastructure for shared asset handling

    Necessary for feature outlined above.

  • Tune server configuration

    The server can always run faster. I need to optimize the nginx caching, the php configuration files, the Percona configuration files. Perhaps, implement Varnish (I don't know if this is worth doing atop the nginx caching). Employ Redis for faster queries. Better cache control integration with each layer of caching, especially Cloudflare. There are others.

  • Create an intelligent way to generate a changelog

    I never figured out a good way to do this that can integrate well with the appcast. Still thinking on it.

  • Website tuning

    Tune Drupal configurations for speed. Optimize Drupal caching. Speed up views.

  • Malicious Workflow reporting

    Create a good system to allow for feedback for unsecure or potentially damaging Workflows. (We already employ virus scanning).

  • Rewrite Workflow objects as stand-alone entities

    Doing so could increase speed and elegance.

  • Improve documentation

    This is an on-going process. Packal was developed to help people find Workflows because they're currently buried in the forums, and there is a lot of redundancy because it's too much work to comb through the near year-long forum archives to find out if a workflow exists. Also, we'd love for Packal to be a repository of development information to help newer Workflow authors to create better Workflows. Again, there is so much in the forums that is hard to find. This can help. We need this part to be created by the community. Let's do that. We also need a better infrastructure to optimize how the documentation is created and displayed. There is always more to do.

  • Create better search utilities

    It's always good to find better ways to make Workflows easier to find. This might include creating entirely different views, tweaking the UI, or tuning the search engines.

  • Tune SEO

    Let's help people find not only Packal but also individual Workflows through search engines. I mean, people might not know how to find this great resource, right?

  • Better Social Media Integration

    Right now, integration with Facebook, Twitter, and Reddit is primative. Making it more robust can help create a better community by showing more people how great Alfred actually is.

  • A better Workflow author experience

    Finding and updating your Workflows isn't as good as it could be. Let's make it better by writing a nicer dashboard for Workflow authors. (Partially done, but by no means good enough to be available.

  • Highlighting Author Contributions

    The content of Packal is user contributed. So let's find better ways to give the Workflow authors much deserved credit.

  • Better app discovery

    Workflows integrate with a myriad of OS X apps, and many users have found new applications because of Workflows written for them. Let's make this easier on the Packal site.

  • Website aesthetics and consistency

    Packal is lacking some consistency for internal reasons. The theme repository didn't receive much attention in the initial development. Let's make that nicer, and let's make Packal prettier overall.
